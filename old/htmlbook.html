Формы

Структура формы

Все элементы формы помещаются в тег <form>. Для него должен быть указан атрибут action, а также method и id. Значением атрибута action является url-адрес страницы-получателя введеных данных при отправки формы на сервер. При использования метода get введенные данные будут добавлены в конец url-адреса, указанного в атрибуте action.

get идеален для:
  • коротких форм (поле ввода для поиского запроса
  • при простом получении данных с веб-сервера(без отправки сведений, добавляемых в базу данных

При ипользования метода post данные отправляются на сервер с помощью так называемых html-заголовков.

post используется в случае, если форма:
  • Позволяет пользователям загружать файлы на сайт
  • Очень длинная
  • Содержит конфидециальные данные (пароли)
  • Добавляет сведения в базу данных или удаляет их оттуда

Если атрибут method не указан, то форма будет отправлена на сервер с помощью метода get. Значения атрибута id используют для различения формы и других элементов разметки страницы. Зачастую этот атрибут используют для проверки заполнения всех обязательных полей формы.

Ввод текста

Элемент input используется для создания компонентов формы нескольких типов.
Значение атрибута type указывает этот тип. Когда атрибуту type присваевается значение text, это значит что будет создано однострочное поле ввода текста.
Атрибут maxlenght используют для ограничения количества символов.
Атрибут size использовали для установки ширины поля ввода текста (но лучше использовать CSS).
Значения атрибута name служат для идентификации элемента формы и отправляются на сервер вместе с введенными данными. Для каждого элемента формы нужно следует указывать данный атрибут.

Переключатели

Используется тег input. Позволяют выбрать только один вариант из предложенных.
Атрибут type равен radio.
Атрибут name отправляется на сервер вместе со значением выбранного переключателя. Когда ответ на вопрос подразумевает выбор одного из нескольких вариантов, все переключатели формирующие список ответов должны иметь одинаковое значение name.
Атрибут value устанавливает значение выбранного переключателя, отправляемое серверу. Значение каждого переключателя должно быть уникальным внутри группы.
Атрибут checked указывает какой, какой из предлагаемых вариантов должен быть выбран по умолчанию при загрузке страницы. Равен значению checked.

Флажки

С помощью флажков можно выбирать несколько вариантов и даже отменять выбор. Используется тег input с type равным checkbox.
Атрибут name отправляется на сервер вместе со значением выбранных вариантов ответа. В данном случае значение name должно быть одинаковым для всех флажков группы.
Атрибут value указывает значение, отправляемое серверу при установке флажка, должно быть уникальным для каждого флажка.

Раскрывающийся список

Используется элемент select, который содержит два и более элементов option.
Атрибут name используется только в элементе select и определяет имя элемента формы, отправляемое на сервер вместе со значением, выбранным посетителем.
Элемент option используется для указания вариантов ответа, которые может выбрать юзер. Текст помещенный между откр. и закр. тегами option, будет выведен на экран как пункт раскрывающегося списка.
Атрибут value используется в элементе option для установки значения, отправляемого на сервер вместе с именем выбранного элемента.

Валидация данных формы

Валидация позволяет гарантировать, что пользователь введет данные в формате, с которым сможет работать сервер.

Она позволяет:

  • уменьшить кол-во операций, выполняемых сервером;
  • увидеть пользователям ошибки в заполнении формы быстрее, чем если бы подобная проверка совершалась на сервере

Примером встроенной системы проверки заполненных данных может служить атрибут required со значением required.

Добавление видео

Используется элемент video с атрибутами:

  • src - значение: url адрес видеофайла
  • poster - позволяет установить изображение, пока видео не запущено
  • width, height
  • preload - указывает браузеру, что нужно делать с видео при загрузке страницы, принимает следующие значения:
    1. none - видео не загружается, пока юзер не нажмет на Воспроизведение
    2. auto - видео загружается по окончании загрузки страницы
    3. metadata - браузер должен лишь собрать метаданые о видео
  • controls - браузер предоставляет собственные элементы управления воспроизведением
  • autoplay
  • loop
